# Job

representation of an ETL job

# runs-on

  • Type: "pyspark" | "pysql" | "jssql"
  • Description: Which runner to use to process this pipeline. pyspark uses pyspark with the dataframe API. pysql uses Python and SQLAlchemy and is suited to an ELT approach. jssql uses Javascript and TypeORM and is suited to an ELT approach

undefined

# steps

  • Type: array

  • Description: List of blocks. These will be run in order, where each step's output is the next step's input, unless the 'inputs' clause is specified

  • Examples:

- id: extract_csv
  uses: extract
  properties:
    source: employee
- id: load_snowflake
  uses: load
  properties:
    table_name: tab
  inputs:
    df: extract/df
